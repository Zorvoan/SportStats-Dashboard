# Log-3

## 3. Týden – Core Logic

### Cíl týdne
Hlavním cílem třetího týdne je napojení externího API pro získávání sportovních dat a implementace základní logiky aplikace. Součástí práce bude také ukládání získaných dat do databáze a jejich následné načítání v aplikaci.

---

## úkoly

### Backend
- Napojení externího Ergast API.
- Implementace endpointů pro získávání dat o F1 závodech.
- Zpracování odpovědí z API.
- Validace přijatých dat.
- Ukládání dat do databáze.
- Optimalizace databázových dotazů.
- Testování správného načítání dat z API.

### Frontend
- Zobrazení dat získaných z API.
- Vytvoření komponent pro výpis výsledků závodů.
- Zobrazení statistik a základních informací o závodech.
- Aktualizace layoutu aplikace.
- Testování komunikace s backendem.

### Databáze
- Ukládání výsledků závodů do databáze.
- Kontrola integrity uložených dat.
- Načítání uložených dat z databáze.
- Testování správnosti ukládání a čtení dat.

## Rozdělení práce v týmu

### Project Manager – Patík
- Koordinace vývoje.
- Kontrola plnění úkolů.
- Organizace týmové práce.

### Backend Developer – Jančík, Kůra
- Napojení Ergast API.
- Zpracování dat z API.
- Ukládání dat do databáze.
- Testování backend logiky.

### Frontend Developer – Poštulka, Zapletal
- Zobrazení dat v aplikaci.

- Testování API komunikace.
- Kontrola správnosti ukládání dat.
- Aktualizace projektové dokumentace.

---

## Výstup týdne

Na konci třetího týdne byla aplikace schopna získávat data z externího Ergast API, ukládat je do databáze a zobrazovat je uživatelům ve frontendové části aplikace. Byla připravena základní logika systému pro další rozšiřování funkcionality.
